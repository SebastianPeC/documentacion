OmniSynch
Sistema ERP de Gestión
de Inventario

Sistema ERP completo y moderno diseñado para PYMEs. Controla productos, inventario, ventas y reportes con un agente inteligente Gemini y un chatbot en Telegram que potencian la automatización del negocio.

Características Principales

Todo lo que necesitas para gestionar tu inventario de forma inteligente.

Autenticación Segura

Sistema de login con JWT y protección contra intentos fallidos con registro en auditoría.

Dashboard Interactivo

Métricas en tiempo real, alertas inteligentes y visualización de datos del negocio.

Gestión de Inventario

Control avanzado de stock, productos, categorías, marcas y variantes con SKUs únicos.

Sistema de Ventas

Punto de venta integrado, órdenes con múltiples items y reportes de transacciones.

Gestión de Usuarios

Roles granulares (admin, gerente, vendedor, almacén) con permisos diferenciados.

Agente Inteligente Gemini

Agente de automatización con Google Gemini que analiza alertas, recomienda acciones y supervisa el inventario.

Chatbot Telegram + Gemini

Bot conversacional en Telegram potenciado con IA para consultar inventario, ventas y recibir alertas.

Interfaz Responsive

Optimizada para desktop y dispositivos móviles con Bootstrap 5.

Stack Tecnológico

Las tecnologías que hacen posible OmniSynch.

N

Node.js con Express.js

Framework HTTP robusto para construir la API REST del sistema.

MySQL 8.0 (Aiven Cloud)

Base de datos relacional en la nube con pool de conexiones y SSL.

JWT (jsonwebtoken)

Autenticación stateless con tokens de 8 horas de expiración.

axios + dotenv

Cliente HTTP para peticiones externas y gestión segura de variables de entorno.

H

HTML5, CSS3, JavaScript ES6+

Tecnologías web nativas para una interfaz rápida y sin dependencias pesadas.

B

Bootstrap 5.3

Framework de diseño responsivo con componentes pre-construidos.

SweetAlert2 + Bootstrap Icons

Notificaciones mejoradas e iconografía consistente en toda la interfaz.

Git + GitHub

Control de versiones con repositorio público en GitHub.

Railway

Plataforma de despliegue en la nube con detección automática de Node.js.

Google Generative AI (Gemini)

Motor de IA utilizado en dos capacidades: el agente de automatización que analiza alertas y genera recomendaciones, y el chatbot de Telegram que responde consultas en lenguaje natural.

node-telegram-bot-api

Integración nativa con la API de Telegram para el bot OmniBot con polling cada 3 segundos.

Instalación Rápida

5 pasos para tener OmniSynch corriendo localmente.

1

Clonar el Repositorio

git clone https://github.com/DevilDantes/OmniS.git
cd OmniS
2

Instalar Dependencias

npm install

⏱ Tiempo estimado: 2-5 minutos

3

Configurar Variables de Entorno

Crea un archivo .env en la raíz. Ve a Variables de Entorno para el contenido completo.

# Windows
type nul > .env
# macOS/Linux
touch .env
4

Verificar Base de Datos

Asegúrate de que las credenciales en .env son correctas. Ve a Base de Datos.

5

Iniciar el Servidor

npm start

✅ Servidor escuchando en el puerto 3000

🤖 OmniBot iniciado

Accede a: http://localhost:3000

¿Cómo obtener los tokens?

🤖

Telegram Bot Token

  1. Abre Telegram y busca @BotFather
  2. Envía /newbot
  3. Sigue las instrucciones (nombre + username)
  4. Copia el token generado en .env
🧠

Google Gemini API Key

  1. Ve a Google AI Studio
  2. Haz clic en "Create API Key"
  3. Selecciona tu proyecto de Google Cloud
  4. Copia la API key en .env

⚠ Esta misma key se utiliza tanto para el agente de automatización como para el chatbot de Telegram.

Base de Datos

Configuración de MySQL local o en la nube.

OmniSynch usa Aiven.io para MySQL en la nube con conexión SSL.

db.js
import mysql from 'mysql2/promise';
export const db = await mysql.createPool({
  host: process.env.DB_HOST,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME,
  port: process.env.DB_PORT,
  ssl: { rejectUnauthorized: false },
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0
});

Instalación de MySQL Local

Windows:

Descarga MySQL Community Server desde dev.mysql.com.

macOS:

brew install mysql
brew services start mysql

Linux (Ubuntu):

sudo apt-get install mysql-server
sudo mysql_secure_installation

Crear la base de datos:

mysql -u root -p

USE omnisynch_inventory;

Docker

Despliegue containerizado con Docker Compose.

Dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
EXPOSE 3000
ENV NODE_ENV=production
CMD ["npm", "start"]
docker-compose.yml
version: '3.8'
services:
  app:
    build: .
    ports: ["3000:3000"]
    environment:
      - DB_HOST=mysql
      - GEMINI_API_KEY=${GEMINI_API_KEY}
      - TELEGRAM_BOT_TOKEN=${TELEGRAM_BOT_TOKEN}
      - JWT_SECRET=${JWT_SECRET}
    depends_on: [mysql]
  mysql:
    image: mysql:8.0
    environment:
      - MYSQL_ROOT_PASSWORD=root_password
      - MYSQL_DATABASE=omnisynch_inventory
    ports: ["3306:3306"]
    volumes: [mysql_data:/var/lib/mysql]
volumes:
  mysql_data:
docker-compose build        # Build
docker-compose up -d        # Ejecutar
docker-compose logs -f app   # Ver logs
docker-compose down          # Detener

Modelo de Datos

SQL Schema (resumen)
CREATE TABLE usuarios (
  id_usuario INT PRIMARY KEY AUTO_INCREMENT,
  nombre VARCHAR(100), correo VARCHAR(100) UNIQUE,
  password_hash VARCHAR(255),
  rol ENUM('admin','gerente','vendedor','almacen'),
  estado ENUM('activo','inactivo','suspendido')
);
CREATE TABLE productos (
  id_producto INT PRIMARY KEY AUTO_INCREMENT,
  nombre VARCHAR(150), precio_base DECIMAL(10,2),
  estado ENUM('activo','inactivo','descontinuado')
);
CREATE TABLE inventario (
  id_inventario INT PRIMARY KEY AUTO_INCREMENT,
  id_variante INT, cantidad_disponible INT DEFAULT 0,
  stock_minimo INT, stock_maximo INT
);
CREATE TABLE ordenes_venta (
  id_orden_venta INT PRIMARY KEY AUTO_INCREMENT,
  numero_orden VARCHAR(50) UNIQUE,
  total DECIMAL(10,2),
  estado ENUM('pendiente','pagada','completada','cancelada')
);
CREATE TABLE auditoria (
  id_auditoria INT PRIMARY KEY AUTO_INCREMENT,
  id_usuario INT, accion VARCHAR(100),
  modulo VARCHAR(50),
  datos_anteriores JSON, datos_nuevos JSON,
  fecha TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Agente de Automatización Gemini

El cerebro inteligente que supervisa y analiza el inventario.

¿Qué hace el agente?

El agente Gemini es un servicio de IA integrado en el backend que recibe contexto completo del negocio, lo procesa con Gemini y devuelve recomendaciones accionables.

📊

Análisis de Alertas

Genera diagnóstico con prioridades de todas las alertas activas.

💡

Recomendaciones de Compra

Sugiere qué pedir, a qué proveedor y con qué urgencia.

📈

Predicción de Demanda

Analiza tendencias de venta para predecir necesidades futuras.

🔍

Diagnóstico de Estado

Detecta anomalías o patrones inusuales en el inventario.

Flujo del Agente

1

Disparador

Vía POST /api/alertas/analizar-ia o POST /api/ia/analizar-ia

2

Recopilación de Contexto

Backend consulta la BD: alertas, stock, ventas recientes.

3

Procesamiento con Gemini

Prompt estructurado enviado a Google Generative AI.

4

Respuesta Inteligente

Análisis estructurado con recomendaciones priorizadas.

5

Registro en Auditoría

Toda interacción queda registrada para trazabilidad.

Chatbot de Telegram

OmniBot — Interfaz conversacional potenciada por Gemini.

Cómo Funciona

El bot corre como servicio background junto al servidor Express. Usa polling cada 3 segundos. Cada mensaje se envía a Google Gemini con contexto del inventario para respuestas inteligentes.

📦Consultar stock
💰Ver ventas y métricas
🚨Recibir alertas de stock
🧠Preguntas en lenguaje natural

Ejemplo de Conversación

¿Cuánto stock de laptops hay?
Hay 15 laptops Dell disponibles. ⚠️ Stock bajo (mín: 20), se recomienda reposición.
¿Hay alertas activas?
2 alertas:
• 🔴 Stock agotado: Cables USB-C
• ⚠️ Stock bajo: Monitores LG (8 uds)
Analiza la situación general
El inventario presenta 2 problemas críticos. Recomiendo pedido urgente de cables USB-C y reposición de monitores esta semana.

Troubleshooting

Solución a problemas comunes.

ErrorCannot find module 'express'

Solución: Ejecuta npm install en la raíz del proyecto.

ErrorECONNREFUSED 127.0.0.1:3000

Solución: Puerto 3000 en uso. Cambia PORT=3001 en .env.

ErrorAccess denied for user

Solución: Verifica credenciales en .env y que MySQL esté corriendo.

ErrorTelegram bot not responding

Solución: Verifica TELEGRAM_BOT_TOKEN en .env. Revisa logs del servidor.

ErrorGEMINI_API_KEY is required

Solución: Verifica API key válida en .env. Confirma cuota en Google AI Studio.

ErrorER_ACCESS_DENIED_ERROR (SSL)

Solución: Asegúrate de tener ssl: { rejectUnauthorized: false } en db.js para Aiven Cloud.

ErrorBot responde vacío o con error

Solución: El prompt puede ser demasiado largo. Revisa logs para el error exacto de Gemini. Reduce el contexto si es necesario.
